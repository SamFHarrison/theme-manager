# `useTheme()`: Tutorial - Build a theme selector

This tutorial walks through adding a theme selector to a React component using the `useTheme()` hook.

By the end, you will have a UI that lets users choose between the three different HBO theme preferences - auto, light and dark.

## What you will learn

In this tutorial, you will learn how to:

- read the user’s saved theme preference
- show the currently resolved theme
- let users choose between auto, light, and dark modes
- update the theme using radio buttons
- confirm the selected theme still works after a page refresh

## Prerequisites

You need:

- a React application,
- the `useTheme()` hook available in your codebase,
- CSS or a design system that responds to the root theme classes.

The hook uses these root classes:

```txt
allow-prefers-dark
obds-color-pref-dark
```

## Step 1: Import the hook

Create or open the component where you want the theme selector to appear.

Import the hook:

```tsx
import { useTheme } from './useTheme';
```

Adjust the import path to match your project structure.

## Step 2: Read theme state from the hook

Inside your component, call `useTheme()`:

```tsx
function ThemeSettings() {
  const { preferredTheme, resolvedTheme, setTheme } = useTheme();

  return null;
}
```

The hook gives you three values.

`preferredTheme` is what the user has chosen.

`resolvedTheme` is the theme currently in effect.

`setTheme` changes the user's preference.

## Step 3: Show the current resolved theme

Replace `return null` with a basic section:

```tsx
function ThemeSettings() {
  const { preferredTheme, resolvedTheme, setTheme } = useTheme();

  return (
    <section>
      <h2>Theme</h2>
      <p>Current resolved theme: {resolvedTheme}</p>
    </section>
  );
}
```

At this point, the component can display whether the app is currently using `dark` or `light`.

This is especially useful when the preferred theme is `auto`, because `auto` can resolve differently depending on the user's system setting.

## Step 4: Add the auto option

Add a radio input for `auto`:

```tsx
<label>
  <input
    type="radio"
    name="theme"
    checked={preferredTheme === 'auto'}
    onChange={() => setTheme('auto')}
  />
  Auto
</label>
```

When the user selects this option, the hook will:

1. store `auto` in `localStorage`,
2. add `.allow-prefers-dark` to the root element,
3. remove `.obds-color-pref-dark`,
4. resolve the active theme from `prefers-color-scheme`.

## Step 5: Add the light option

Add a radio input for `light`:

```tsx
<label>
  <input
    type="radio"
    name="theme"
    checked={preferredTheme === 'light'}
    onChange={() => setTheme('light')}
  />
  Light
</label>
```

When the user selects this option, the hook removes both theme classes from the root element.

That makes the resolved theme `light`.

## Step 6: Add the dark option

Add a radio input for `dark`:

```tsx
<label>
  <input
    type="radio"
    name="theme"
    checked={preferredTheme === 'dark'}
    onChange={() => setTheme('dark')}
  />
  Dark
</label>
```

When the user selects this option, the hook adds `.obds-color-pref-dark` to the root element and removes `.allow-prefers-dark`.

That makes the resolved theme `dark`.

## Step 7: Put the component together

Your finished component should look like this:

```tsx
import { useTheme } from './useTheme';

export function ThemeSettings() {
  const { preferredTheme, resolvedTheme, setTheme } = useTheme();

  return (
    <section>
      <h2>Theme</h2>
      <p>Current resolved theme: {resolvedTheme}</p>

      <label>
        <input
          type="radio"
          name="theme"
          checked={preferredTheme === 'auto'}
          onChange={() => setTheme('auto')}
        />
        Auto
      </label>

      <label>
        <input
          type="radio"
          name="theme"
          checked={preferredTheme === 'light'}
          onChange={() => setTheme('light')}
        />
        Light
      </label>

      <label>
        <input
          type="radio"
          name="theme"
          checked={preferredTheme === 'dark'}
          onChange={() => setTheme('dark')}
        />
        Dark
      </label>
    </section>
  );
}
```

## Step 8: Try the behaviour

Open the page that renders `ThemeSettings` and open up your browser dev tools to see local storage and the classes on the root element.

Choose **Dark**. The root element should receive `obds-color-pref-dark`

Choose **Light**. The root element should have neither theme class

Choose **Auto**. The root element should receive `allow-prefers-dark`

If your system is currently using dark mode, `resolvedTheme` should be `dark`. If your system is currently using light mode, `resolvedTheme` should be `light`. You can also use browser dev tools to emulate switching browser theme preferences.

## Step 9: Refresh the page

After choosing a theme, refresh the page.

The selected preference should still be used because the hook stores it in:

```txt
localStorage['hbo-theme-preference']
```

If no stored preference exists, the hook defaults to `auto`.

## What you have learned

You have built a theme selector that:

- stores the user's preference,
- applies the correct root classes,
- follows system colour scheme in `auto` mode,
- exposes the actual resolved theme,
- survives page reloads when storage is available.
